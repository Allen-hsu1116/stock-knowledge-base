停損停利是交易策略的關鍵環節，但一般停損停利往往無法應對市場波動，導致獲利過早出場或停損過大。

## 移動停利停損簡介

移動停利（Trailing Stop），又稱移動鎖利，也有人稱為動態停利，主要是隨著獲利創新高後回檔特定 % 數或價格，來進行停利出場操作，是一種能夠避免大量獲利回吐下仍能守住獲利的出場策略。

移動停利的優勢在於能夠根據市場變化來自動調整，會隨著獲利創新高而逐步調整停利點的一種停利策略。

## 策略實作邏輯

1. 取得策略開始後到最近一個交易日的最高價格
2. 盤中即時價格高於歷史最高價時，最高價將被即時價格取代
3. 當即時價格 <= 最高價 × (1 - 停利比例)，觸發停利出場

## 程式碼核心邏輯

```python
# 移動停損停利判斷
def run_strategy(self, qty):
    if self.now_price is not None:
        # 盤中需判斷目前價位是否為更新價位
        if self.now_price >= self.high_price:
            self.high_price = self.now_price
        
        # 達到停利就進行平倉操作
        if self.now_price <= self.high_price * (1 - self.stopProfit_ratio):
            # 平倉
            self.sell_total_shares(qty)
```

## 進階變化

- 最高點向下回檔 n% 就賣出一半的個股，進行分批出場操作
- 可結合均線、ATR等技術指標作為移動停利的基準
- 可加入成交量確認條件，避免假突破觸發停利
