## Renko 圖表基礎

Renko來自日文「renga」（磚塊），完全忽略時間，只在價格移動超過設定磚塊大小時畫新磚。

核心邏輯：
- 選擇磚塊大小（如$1、10點、1×ATR）
- 收盤價≥磚塊大小以上 → 畫綠磚
- 收盤價≤磚塊大小以下 → 畫紅磚
- 時間無關：高波動時一分鐘一磚，低波動時數小時一磚

## Renko vs K線圖對比

| Renko | K線 |
|-------|-----|
| 過濾噪音，每塊磚=有意義的價格變動 | 每個時間段都畫一根K線，盤整看起來亂 |
| 支撐壓力更清晰 | 影線扭曲支撐壓力 |
| 沒有時間軸，專注價格行為 | 包含時間但有干擾（新聞K線、午盤盤整） |
| 突破進場較晚（需完整磚塊確認） | 突破訊號更快但假突破更多 |
| 適合趨勢追蹤和波段交易 | 適合剝頭皮和成交量分析 |

## 磚塊大小選擇三方法

### 3.1 固定值（Manual）
選一個整數（如EURUSD 10點、AAPL $1）。回測簡單，但無法適應波動變化。

### 3.2 ATR乘數（Adaptive）
ATR(14)×1.0到1.5作為磚塊大小。自動適應市場波動，切換市場不用手動調整。低波動用ATR×1、高波動用ATR×1.5。

### 3.3 Walk-Forward回測法（Hybrid）
從ATR×1.2開始，回測後±10%微調直到淨利/最大回撤合理且權益曲線平滑。鎖定後每月或重大波動事件後重測。

## TradingView 設定步驟

1. 開啟圖表
2. 點選圖表類型下拉選單 → 選Renko
3. 選Traditional或ATR磚塊
4. 設定磚塊大小（如1 ATR）
5. ⚙️設定中可啟用Wick（影線）
6. 加入指標：Supertrend、200 EMA、RSI等

## 常見錯誤

1. **磚塊太小** → 圖表像K線圖，訊號太頻繁
2. **對抗大時間框架趨勢** → 永遠先看大週期方向
3. **忽略風險管理** → Renko過濾噪音不過濾虧損
4. **指標疊加過多** → 一兩個確認工具就夠（EMA 200 + Supertrend）
5. **波動變化後沒調整磚塊大小** → 財報季、FOMC後必須重新設定

## Renko 進階策略搭配

- Renko看趨勢和支撐壓力 → 切回K線找進場型態
- Renko+Supertrend：趨勢確認+進出場
- Renko+RSI/MACD：指標在Renko上更清晰
- Renko+200EMA：多空分界+趨勢追蹤

## 適用時間框架

- 波段交易：日線或4小時
- 日內交易：5分鐘+ATR磚塊
- 加密貨幣：日線ATR×1自動適應
