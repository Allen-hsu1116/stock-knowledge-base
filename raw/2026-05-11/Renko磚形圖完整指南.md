## Renko Charts 是什麼

Renko（練行）來自日文「煉瓦」（renga），意思是「磚塊」。

Renko圖表只看價格、不看時間。每當價格移動超過設定的磚塊大小（brick size），就畫一塊新磚。上漲畫綠磚（或白磚）、下跌畫紅磚（或黑磚）。

核心邏輯：
- 設定磚塊大小（如1美元、10點、或1倍ATR）
- 收盤價≥前磚收盤價+磚塊大小 → 畫新綠磚
- 收盤價≤前磚收盤價-磚塊大小 → 畫新紅磚
- 時間無關：一塊磚可能一分鐘形成，也可能數小時

## 磚塊大小設定

### Traditional 固定值
手動設定固定數值（如50點、$1）。簡單直觀但不適應波動變化。

### ATR 自適應
用ATR(14)作為磚塊大小基礎，自動適應不同市場和波動環境。ATR×1.0到ATR×1.5是常見範圍。低波動用ATR×1，高波動用ATR×1.5。

### Walk-Forward 回測法
從ATR×1.2開始，回測後±10%微調，直到淨利/最大回撤比合理且權益曲線平滑。每月或重大事件後重新測試。

## 時間框架影響

時間框架仍影響Renko：
- 日線：ATR大→磚塊大
- 小時線：ATR小→磚塊小
- 5分鐘：更小的ATR→更小的磚塊

用ATR設定就不用每換市場手動調整磚塊大小。

## Renko 的優勢

1. **趨勢辨識超清晰**：連續綠磚=上漲趨勢、連續紅磚=下跌趨勢、交替=盤整
2. **過濾市場噪音**：移除K線圖中的長上下影線、小波動、新聞事件造成的雜訊
3. **支撐壓力更清楚**：沒有影線干擾，水平支撐壓力一目了然
4. **型態辨識更容易**：頭肩頂/底、雙頂/雙底在Renko圖上更清晰
5. **避免過早出場**：設定「2塊反轉磚」作為出場條件，避免被正常回測洗出場

## Renko 的缺點

1. **突破進場較晚**：需要完整磚塊確認，可能錯過突破的第一段
2. **不顯示時間維度**：無法判斷盤整持續多久
3. **不顯示成交量**：需要額外指標（OBV等）
4. **不顯示缺口**：隔夜缺口會變成一系列磚塊而非單一跳空
5. **盤整市仍然會被洗**：磚塊太小等於沒過濾、太大則遲鈍

## Renko + 均線策略

用20MA搭配Renko：
- 價格在MA下方 → 做空
- 價格在MA上方 → 做多
- 盤整中綠磚不突破MA → 續持空單
- 反轉磚穿越MA → 出場訊號

Renko+MA的核心優勢：讓你持有趨勢更久，不會被正常回調洗出場。

## Renko 實戰搭配

- Renko看趨勢和支撐壓力 → 切回K線找進場型態
- Renko+Supertrend：趨勢確認+進出場訊號
- Renko+RSI/MACD：指標在Renko上更清晰
- 磚塊大小是唯一最重要的參數，太大太小都不行

## 常見錯誤

1. 磚塊太小 → 圖表像K線圖，訊號太頻繁
2. 對抗大時間框架趨勢 → 永遠先看大週期方向
3. 忽略風險管理 → Renko過濾噪音不過濾虧損
4. 指標疊加過多 → 一兩個確認工具就夠
5. 波動變化後沒調整磚塊大小 → 財報季、FOMC後必須重新設定

## 適用場景

- 趨勢追蹤和波段交易：Renko最擅長
- 日內交易：5分鐘Renko+ATR磚塊
- 加密貨幣：波動大，ATR磚塊自適應
- 不適合：剝頭皮、成交量分析、時間相關策略
