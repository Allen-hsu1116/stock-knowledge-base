# 股票知識庫規則說明

## 這個知識庫是什麼

一份關於「股票操作」的個人知識庫，目標是成為股票操作大師。

核心方向：
- 技術分析：K線、均線、指標、型態
- 基本面分析：財報、估值、產業
- 籌碼面分析：三大法人、大戶、融資券
- 操作策略：當沖、波段、停損停利
- 風險管理：倉位控制、心理建設

## 資料夾結構

- **raw/**：原始素材暫存區，AI 不得修改此資料夾內的任何檔案
- **wiki/**：整理後的知識庫，由 AI 全權維護，使用者不手動編輯
- **outputs/**：AI 產出的報告、回答、分析歸檔

## Wiki 維護規則

### 檔案結構
- 每個主題建立一份獨立的 `.md` 檔案
- 依照性質放入對應子資料夾（技術分析/基本面分析/籌碼面分析/操作策略/風險管理）
- 檔名使用中文，格式：`主題名稱.md`

### 每篇 Wiki 格式
```markdown
# 主題名稱

> 一句話摘要

## 核心概念
[說明這個概念是什麼]

## 實戰應用
[如何實際使用]

## 注意事項
[陷阱、誤區、限制]

## 相關主題
- [[相關主題A]]
- [[相關主題B]]

## 來源
- [文章標題](raw/日期/檔名.md)
```

### 交叉連結
- 相關主題之間用 `[[主題名稱]]` 格式互相連結
- 自動發現新連結機會，更新相關文章

### INDEX.md 維護
- wiki/ 中維護一份 INDEX.md
- 列出所有主題，依分類組織
- 格式：
```markdown
# 股票知識庫索引

## 技術分析
- [[KD指標]] - 隨機指標，判斷超買超賣
- [[RSI背離]] - 相對強弱指標背離訊號
- ...

## 基本面分析
- [[杜邦分析]] - 拆解 ROE 的五因子模型
- ...

...
```

## 編譯流程

當 raw/ 新增素材時：
1. 讀取新素材內容
2. 識別涉及的主題
3. 為每個主題建立或更新 wiki 文章
4. 更新相關文章的交叉連結
5. 更新 INDEX.md

## 提問與回饋

使用者提問後：
1. 搜尋 wiki/ 相關內容
2. 綜合回答
3. 將回答存入 outputs/
4. 如果發現新洞察，更新相關 wiki 文章

## 健康檢查

定期執行：
- 掃描文章間的矛盾
- 找出被提及但沒解釋的主題（紅字連結）
- 列出缺乏 raw/ 來源支持的論點
- 建議可補充的文章

## Quartz 網站部署

知識庫透過 Quartz v4 部署到 GitHub Pages。

- **Repo**: https://github.com/Allen-hsu1116/stock-knowledge-base
- **網站**: https://allen-hsu1116.github.io/stock-knowledge-base/
- **本地 Quartz 專案**: `~/.openclaw/workspace/stock-quartz-site/`
- **同步腳本**: `stock-quartz-site/sync-wiki-to-quartz.sh`
- **部署方式**: Push 到 main → GitHub Actions 自動 build + deploy

### 更新流程
1. 整理完知識庫後，執行 `sync-wiki-to-quartz.sh`
2. `cd stock-quartz-site && npx quartz sync`
3. GitHub Actions 自動部署
